Apuntes básicos de HTML

Elementos básicos de HTML

  • <h1> a <h6>: Encabezados de diferentes niveles, siendo <h1> el más importante.
  • <p>: Define un párrafo de texto.
  • <a>: Crea un hipervínculo a otra página o recurso.
  • <img>: Inserta una imagen en la página web.
  • <ul> y <ol>: Listas no ordenadas (con viñetas) y ordenadas (numeradas).
  • <li>: Define un ítem dentro de una lista.
  • <div>: Contenedor genérico para agrupar elementos y aplicar estilos o scripts.
  • <span>: Contenedor en línea para aplicar estilos a una parte del texto.
  • <form>: Define un formulario para la entrada de datos del usuario.
  • <input>: Campo de entrada dentro de un formulario (texto, botón, casilla de verificación, etc.).
  • <button>: Botón interactivo que puede activar acciones.
  • <table>: Crea una tabla para organizar datos en filas y columnas.
  • <tr>: Define una fila dentro de una tabla.
  • <td>: Define una celda dentro de una fila de tabla.
  • <th>: Define una celda de encabezado dentro de una tabla.
  • <meta>: Proporciona metadatos sobre el documento, como la codificación de caracteres y la descripción.
  • <link>: Vincula recursos externos, como hojas de estilo CSS.
  • <script>: Inserta o referencia código JavaScript para añadir interactividad.
  • <style>: Define estilos CSS directamente dentro del documento HTML.

Buenas prácticas en HTML

Para escribir un código HTML limpio y eficiente, es importante seguir ciertas buenas prácticas. Primero, siempre utiliza una estructura semántica adecuada, empleando las etiquetas correctas para cada tipo de contenido (por ejemplo, <article> para artículos, <nav> para navegación). Segundo, mantén el código bien organizado e indentado para mejorar la legibilidad. Tercero, utiliza atributos descriptivos como "alt" en imágenes para mejorar la accesibilidad. Cuarto, evita el uso excesivo de etiquetas <div> y <span>, optando por etiquetas más específicas cuando sea posible. Quinto, asegúrate de que tu HTML sea compatible con diferentes navegadores y dispositivos mediante pruebas regulares. Sexto, minimiza el uso de estilos en línea y scripts dentro del HTML, prefiriendo archivos externos para CSS y JavaScript. Séptimo, valida tu código utilizando herramientas como el validador de W3C para detectar errores y asegurar el cumplimiento de los estándares web. Finalmente, mantente actualizado con las últimas especificaciones de HTML y las mejores prácticas recomendadas por la comunidad web.

El elemento Imagen

El elemento <img> en HTML se utiliza para insertar imágenes en una página web. Es una etiqueta auto-cerrada que requiere al menos el atributo "src" para especificar la ruta de la imagen y el atributo "alt" para proporcionar un texto alternativo que describe la imagen, lo cual es crucial para la accesibilidad y el SEO. Además, se pueden usar otros atributos como "width" y "height" para definir las dimensiones de la imagen, aunque es recomendable controlar el tamaño mediante CSS para mayor flexibilidad. También es posible utilizar atributos como "title" para mostrar un tooltip al pasar el cursor sobre la imagen. El elemento <img> no tiene una etiqueta de cierre, y su sintaxis básica es: <img src="ruta/de/la/imagen.jpg" alt="Descripción de la imagen">. Es importante optimizar las imágenes para la web, asegurándose de que tengan un tamaño adecuado y un formato apropiado (como JPEG, PNG o WebP) para mejorar los tiempos de carga y la experiencia del usuario.

Enlaces

Los enlaces en HTML se crean utilizando la etiqueta <a> (ancla). Esta etiqueta permite vincular a otras páginas web, archivos, direcciones de correo electrónico o incluso a secciones específicas dentro de la misma página. El atributo principal de la etiqueta <a> es "href", que especifica la URL del destino del enlace. Por ejemplo, <a href="https://www.ejemplo.com">Visitar Ejemplo</a> crea un enlace que lleva al usuario a "https://www.ejemplo.com". Además, se pueden utilizar otros atributos como "target" para definir cómo se abrirá el enlace (por ejemplo, en una nueva pestaña con target="_blank") y "title" para proporcionar información adicional que aparece al pasar el cursor sobre el enlace. Es importante asegurarse de que los enlaces sean descriptivos y accesibles, utilizando texto claro y relevante para mejorar la experiencia del usuario y la optimización en motores de búsqueda (SEO).

Formularios

Los formularios en HTML se crean utilizando la etiqueta <form>, que actúa como un contenedor para diversos elementos de entrada que permiten a los usuarios enviar datos a un servidor. Dentro de un formulario, se pueden incluir diferentes tipos de campos de entrada, como <input> (para texto, contraseñas, botones, casillas de verificación, etc.), <textarea> (para áreas de texto más grandes), <select> (para menús desplegables) y <button> (para botones interactivos). El atributo "action" de la etiqueta <form> especifica la URL a la que se enviarán los datos del formulario, mientras que el atributo "method" define el método HTTP utilizado para el envío (generalmente "GET" o "POST"). Es importante incluir etiquetas <label> para cada campo de entrada para mejorar la accesibilidad y la usabilidad del formulario. Además, se pueden implementar validaciones tanto en el lado del cliente (usando atributos HTML5 o JavaScript) como en el lado del servidor para asegurar que los datos ingresados sean correctos y seguros antes de procesarlos.

Tablas

Las tablas en HTML se crean utilizando la etiqueta <table>, que sirve como contenedor para organizar datos en filas y columnas. Dentro de una tabla, las filas se definen con la etiqueta <tr> (table row), mientras que las celdas de datos se crean con la etiqueta <td> (table data) y las celdas de encabezado con la etiqueta <th> (table header). Es común utilizar <th> en la primera fila o columna para proporcionar títulos descriptivos que ayuden a identificar el contenido de las celdas correspondientes. Además, se pueden aplicar atributos como "border", "cellpadding" y "cellspacing" para controlar el estilo y el espaciado de la tabla, aunque es recomendable usar CSS para un mayor control visual. Las tablas son útiles para presentar datos tabulares de manera clara y estructurada, pero deben usarse con moderación para evitar complicaciones en el diseño responsivo y la accesibilidad. Es importante también considerar el uso de elementos semánticos adicionales, como <caption> para añadir un título a la tabla y <thead>, <tbody> y <tfoot> para organizar mejor el contenido de la tabla.
